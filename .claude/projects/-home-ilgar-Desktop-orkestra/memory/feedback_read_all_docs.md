For product, ideation, or cross-cutting conversations, read ALL docs before
responding. Do not ask the user questions that are already answered in the
repo's own documentation.

**Why:** User called out that I asked "where does design happen?" when
AGENTS.md principle #10 already says "Figma is registered truth." The repo
is designed to be agent-native — the structure IS the answer.

**How to apply:** When the conversation is about product shaping, brainstorming,
architecture, or design — load everything listed in the AGENTS.md full-context
rule before speaking. For focused implementation, the thin paths in day-one.md
are fine.
